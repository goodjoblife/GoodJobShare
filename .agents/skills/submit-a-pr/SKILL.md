name: submit-a-pr
description: when user what to submit a pr
---

1. identity current branch, if your are on master, check a new branch
2. use `git status` to check files are commited, it not, guide user to commit
3. use `gh` command to creat a PR in github.com/goodjoblife/GoodJobShare
   - the title should be short to summarize the change of between branch and master
   - the description should follow the `.github/PULL_REQUEST_TEMPLATE.md`
4. show the link to user
